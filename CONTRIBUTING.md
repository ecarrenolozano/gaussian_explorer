# Contributing

1. Start from an approved user story and technical-readiness assessment.
2. Create or update the feature implementation plan and TDD test list.
3. Add tests before or alongside implementation.
4. Run all local quality checks.
5. Update traceability and relevant SDLC documentation.
6. Obtain validation and acceptance approval before integration.

Use `repository_templates/change_request.md` when opening a pull request or merge request.
