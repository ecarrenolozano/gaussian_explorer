# Interface and Accessibility Product Overlay

Apply only to product-facing behavior:

- Are supported user contexts, devices, platforms, orientations, input methods, and interrupted flows bounded where they affect acceptance?
- Are loading, empty, error, permission-denied, offline, resume, and stale-content states represented when material?
- Can intended users complete the outcome with relevant keyboard, assistive, touch, voice, or alternative input?
- Are focus, labels, status changes, instructions, errors, contrast, motion, timing, and recovery outcomes testable where required?
- Are notifications, consent, localization, time zones, formats, text expansion, and language behavior clear when in scope?
- Are accessibility or inclusive-design obligations tied to realistic user workflows rather than generic compliance claims?
