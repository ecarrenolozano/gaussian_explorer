# Platform and Write Safety

- Follow repository issue forms and contribution rules.
- Show every proposed tracker write completely in chat before requesting approval.
- Use machine-readable lineage in the issue body when the platform supports free-form text.
- Use `templates/issue-template-repository.md` unless the repository has a stricter native issue form.
- After a write, retrieve the issue and verify identifier, URL, body, lineage, labels, milestone, assignees, and state.
- If the response is ambiguous, search for the lineage marker before retrying.
- Never claim publication without verification.
