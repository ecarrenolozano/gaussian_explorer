# Evidence and Status

For each requirement, risk, or test target, record:

- verification status: `passed`, `failed`, `not-run`, `blocked`, or `not-applicable`;
- evidence type: automated test, static analysis, inspection, external report, or other technical evidence;
- evidence reference;
- test category;
- artifact and environment;
- notes and blocking dependencies.

Do not classify manual stakeholder validation as automated test evidence. Record `manual-validation-required` and route the item to Skill J when appropriate.
