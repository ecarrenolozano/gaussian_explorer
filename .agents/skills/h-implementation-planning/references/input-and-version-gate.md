# Input and Version Gate

Require:

- repository issue identifier and URL;
- linked source story or approved spike candidate;
- current story version;
- current architecture version;
- applicable technical review identifier;
- valid technical approval for the selected story;
- repository or code baseline used for planning.

Do not plan from an unlinked or diverged issue. Reconcile it through Skill G first.

A changed story version, architecture version, technical review, or materially changed issue scope makes an earlier plan stale.
