# Persistence and Readiness

Use plan versions such as `1.0`, `1.1`, and `2.0` according to repository convention. Bind each plan to exact issue IDs, story version, architecture version, technical review, and code baseline.

Statuses:

- `ready`: implementation can begin;
- `ready-with-open-items`: non-blocking items remain visible;
- `changes-required`: plan must be revised;
- `blocked`: implementation cannot begin.

A material scope, design, sequencing, migration, or verification change creates a new plan version. Editorial corrections do not require a new version when meaning is unchanged.
