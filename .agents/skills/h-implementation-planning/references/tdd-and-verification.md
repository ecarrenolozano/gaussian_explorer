# Developer Tests and Verification

For each implementation increment, define:

- observable objective;
- failing or missing developer test when test-first work is appropriate;
- implementation change;
- refactoring or cleanup;
- affected files;
- verification command or method;
- completion condition.

Developer tests may include focused unit, component, contract, or local integration tests. Skill I later owns broader integration, end-to-end, regression, and other release-level automated verification.
