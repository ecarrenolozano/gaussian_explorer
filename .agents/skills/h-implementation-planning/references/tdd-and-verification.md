# Test-Driven Development and Verification

For suitable implementation increments:

1. identify the smallest observable behavior;
2. define the first failing developer test;
3. implement the minimum change to pass;
4. refactor while preserving behavior;
5. add integration coverage for important boundaries.

Developer tests primarily include unit and focused component tests. Integration tests may be planned when they are tightly coupled to the increment.

Skill I remains responsible for the broader integration, end-to-end, regression, and other release-oriented automated verification strategy. Do not duplicate that complete strategy here.
